今天先把仓库完整过了一遍，结论很清楚：项目本身已经从“最小 TCP server”进到“协议 + 非阻塞连接管理”的阶段，但文档和目录管理没有同步跟上。

当前代码层面的主线是：

## 本周项目总结

### 1. 这周 TinyIM 做成了什么

按时间线看，本周项目推进是比较清楚的：

1. Day 01：把项目骨架搭起来，先有 `src/`、`include/`、`docs/`、`conf/`
2. Day 02：把最小 server 拆成 `TcpServer` 和 `Logger`，服务端不再因为一次连接结束就退出
3. Day 03：加入 `Config`，把端口和日志级别从硬编码改成配置驱动
4. Day 04：开始补协议意识，认识到 TCP 是字节流，不能把一次 `recv()` 当一条消息
5. Day 05：把监听 fd 和连接 fd 全部改成非阻塞，开始维护连接状态、输入缓冲区、输出缓冲区
6. Day 06：回头整理文档、目录和决策记录，让仓库对别人也可读

如果只看“有没有更多功能”，这一周看起来不像功能爆发；但如果看“网络服务端基础有没有一步步做对”，进展是扎实的：
- 已经不再停留在演示级别的阻塞式 echo server
- 已经开始按真实服务端的思路处理消息边界和 I/O 语义
- 已经具备继续引入 `epoll` 的基础

### 2. 本周做对的关键点

- 先拆模块，再加功能。没有把所有逻辑继续塞回 `main.cpp`
- 配置从一开始就独立出来，避免后面每次加参数都改源码
- 在 Day 04 及时意识到“TCP 没有消息边界”这个核心事实
- 在 Day 05 没急着直接上 `epoll`，而是先把非阻塞读写语义搞明白

这几个选择本质上都在做同一件事：先把抽象边界和运行语义打稳，再继续往前堆功能。

### 3. 本周还没做完的部分

- 还没有优雅退出和信号处理

## 本周算法题型总结

这部分先按“基础训练周”的思路做复盘，不强行列题号，重点总结题型和通病。

### 1. 本周最该继续打牢的题型

#### 数组与双指针

这是最基础、但也最容易写出边界 bug 的一类。

常见考点：
- 有序数组上的左右指针
- 原地去重 / 原地移除
- 滑动窗口
- 快慢指针

这类题最容易错的地方不是不会想，而是：
- 循环结束条件写错
- 指针移动顺序写反
- 先更新答案还是先收缩窗口没有想清楚

复盘建议：
- 每次都先写出“窗口含义”
- 明确哪个变量表示答案，哪个变量表示状态
- 纸上先过一组最小样例，再落代码

#### 链表模拟

链表题的难点通常不在算法复杂度，而在指针改动顺序。

本周这类题如果出错，大概率都集中在：
- 头结点变更时没有 dummy node
- 修改 `next` 之前没有先保存后继
- 以为自己处理了空链表，实际只处理了单节点

复盘建议：
- 只要涉及删除、反转、分组操作，优先上 dummy node
- 每次改指针前，先口头说清楚“断开谁、接上谁”

#### 栈与队列

这类题容易“思路知道，代码写乱”。

常见表现：
- 把栈题写成 if，而不是 while，导致无法连续弹出
- 单调栈里分不清“维护什么单调性”
- BFS 队列层序遍历时，当前层和下一层混在一起

复盘建议：
- 栈题先回答：栈里存什么，入栈条件是什么，弹栈条件是什么
- BFS 先回答：是求最短步数，还是单纯遍历所有节点

#### 二分查找

二分属于“写起来像模板，错起来也像模板”的典型题型。

高频错误：
- 区间定义不统一，`[l, r]` 和 `[l, r)` 混用
- `mid` 命中后不知道该收哪一边
- 搜索目标从“找值”变成“找边界”后，判断条件没有同步变化

复盘建议：
- 每一题都先写一句：我当前维护的是闭区间还是左闭右开区间
- 写完后立即用 1 个元素、2 个元素、全不命中三个样例手推

### 2. 本周错题暴露出来的弱点

#### 弱点一：边界意识不稳定

很多题不是主思路错，而是：
- 空输入没处理
- 单元素没处理
- 最后一步多走或少走一格

说明当前最大问题不是“不会套路”，而是“实现不够稳”。

改进方向：
- 每题提交前固定检查空、单个、全相同、严格递增、严格递减这几类样例

#### 弱点二：状态定义不够清楚就开始写

尤其在窗口、DP、单调栈题里，如果没先想清楚“这个变量究竟表示什么”，代码就会越写越绕。

改进方向：
- 先用一句中文定义状态，再写代码
- 如果一句话说不清状态，说明还不该落实现

#### 弱点三：把“看懂题解”误当成“掌握题型”

这是非常常见的问题：
- 看题解时觉得自己懂了
- 关掉题解后，变量和边界又写不出来

改进方向：
- 题解只能看思路，不看最终代码抄写
- 第二天必须脱离题解重写一次
- 错题至少做“当天改一次，隔天重写一次”

#### 弱点四：基础题复盘不够系统

如果本周做题是零散推进，就容易出现：
- 做过双指针，但没有沉淀模板
- 做过链表，但没有总结 dummy node 的适用场景
- 做过二分，但不同边界题写法还在摇摆

改进方向：
- 不要只存题目，要存“题型模板 + 易错点”
- 每类题保留 2 到 3 道代表题，反复回刷

## 今日完成

### 1. 增加配置模块 `Config`
新增了配置类，用于读取类似 `key=value` 的配置文件。

当前已经支持：
- 读取整数配置，如 `port`
- 读取字符串配置，如 `log_level`
- 跳过空行和注释行
- 当 key 不存在或类型转换失败时，返回默认值

对应文件：
- `include/Config.h`
- `src/Config.cpp`
- `conf/server.conf`

当前配置示例：

```conf
port=9999
log_level=INFO
```

### 2. `main` 中接入配置
服务端启动时会先读取配置文件，再把配置传给模块：
- `port` 用于决定监听端口
- `log_level` 用于初始化日志级别

这样相比硬编码更方便：
- 想换端口时不用改源码
- 想控制日志输出级别时也不用重新设计接口

### 3. CMake 改成 target 级 include
当前 `CMakeLists.txt` 使用的是：

```cmake
target_include_directories(TinyIM PRIVATE
    ${CMAKE_SOURCE_DIR}/include
)
```

这样做的好处是：
- 头文件搜索路径绑定到具体目标 `TinyIM`
- 比全局 `include_directories(...)` 更清晰
- 项目变大后更方便维护依赖关系

## Linux 基础笔记

### 1. `ss`：查看 socket / 端口监听情况
常用命令：

```bash
ss -ltnp
```

含义：
- `-l`：只看监听中的 socket
- `-t`：TCP
- `-n`：直接显示数字，不做域名/服务名解析
- `-p`：显示对应进程

适合用来确认：
- 某个端口是否真的在监听
- 是哪个进程占用了这个端口

例如排查 `9999` 端口：

```bash
ss -ltnp | grep 9999
```

### 2. `lsof`：通过文件视角查看端口被谁占用
常用命令：

```bash
lsof -i :9999
```

作用：
- 查看哪个进程正在使用 `9999` 端口
- 适合排查“端口被占用，服务端启动失败”这类问题

### 3. `ps`：查看进程是否还活着
常用命令：

```bash
ps -ef | grep TinyIM
```

作用：
- 看服务端进程是否存在
- 查看进程 pid，方便后续进一步分析或结束进程

### 4. `top`：动态查看系统和进程状态
直接执行：

```bash
top
```

作用：
- 看 CPU、内存占用
- 观察服务端进程在运行时的资源情况
- 程序异常卡住时，可快速判断是不是资源问题
